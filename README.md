# QA Professional Portfolio

A modern, high-performance portfolio website showcasing my work as a Quality Assurance Engineer and my journey in software testing. Designed with an emphasis on clean UI, speed, and clear documentation.

[**View Live Portfolio**](https://syefdi.github.io)

## Features

- **Clean & Professional UI:** Built with a dark-mode glassmorphism aesthetic inspired by modern enterprise software.
- **Fast & Lightweight:** Operates as a pure HTML file utilizing Tailwind CSS via CDN, with zero build steps required.
- **Comprehensive QA Documentation:** Includes real-world examples of Bug Reports, Smoke Tests, Test Cases, and Metrics Dashboards.
- **Fully Responsive Layout:** Optimized viewing experience across desktop, tablet, and mobile devices.

## Project Structure

The repository is kept intentionally lean and clean, focusing strictly on content rather than complex build processes.

```text
1portfolio/
├── index.html              # Main portfolio page (includes inline JS & Tailwind config)
├── README.md               # Project documentation
├── PRODUCT.md              # Product strategy and brand guidelines
├── DESIGN.md               # Design system documentation
│
├── docs/                   # QA Documentation Examples
│   ├── bug-report.html     # Standard bug reporting format with examples
│   ├── smoke-test.html     # Critical happy path module checklist
│   ├── test-cases.html     # Comprehensive test scenarios
│   └── metrics.html        # Summary of defect density and pass rate metrics
│
└── assets/                 # Static assets
    └── images/             # Profile photos, favicons, and other imagery
```

## Tech Stack

- **HTML5** - Semantic and accessible markup.
- **Tailwind CSS (CDN)** - Utility-first styling with custom configuration.
- **Vanilla JavaScript** - Lightweight, inline scripts for interactive elements (scroll reveal, mobile navigation, cursor spotlight).
- **Google Fonts** - *Plus Jakarta Sans* for modern readability, *Fira Code* for monospace accents.

## Local Development

Getting the portfolio running locally is incredibly simple:

1. Clone the repository: `git clone https://github.com/Syefdi/Syefdi.github.io.git`
2. Open `index.html` directly in any modern web browser.
3. No `npm install` or build process is required!

## Deployment

The portfolio is deployed via **GitHub Pages**. Any code merged into the `main` branch is automatically published to the live site.

## Documentation Artifacts

This repository serves not only as a visual portfolio but also as a demonstration of technical writing capabilities. The `docs/` folder contains comprehensive English documentation that recruiters and hiring managers can review to assess my QA methodology.

## Maintenance

- **Adding New Projects:** Edit `index.html` and add a new card to the Projects section grid.
- **Updating Copy:** The repository is centralized in `index.html`. All textual content, links, and structure can be modified there.
- **Styling Changes:** Tailwind configuration (colors, fonts, etc.) is located at the top of `index.html` within the `<script id="tailwind-config">` tag.
