Test Case Implementation Examples

check_circle TC-001 - Positive Case: Risk Input Successfully Saved
Test IDTC-ERM-RI-001
ScenarioRisk successfully created and moved to Need Follow Up tab
ModuleRisk Input - BAU
Pre-conditionUser is logged in as Risk Owner with active reference. Current control evaluation is available.
Test Steps
  1. Click Risk Input menu.
  2. Fill all mandatory fields: Risk Description, Inherent Risk (Likelihood + Severity), Current Control Evaluation = "Require to Improve", Action = "Reduce", Residual Risk Plan, Objective, BSC.
  3. Add 1 mitigation with a valid PIC, Due Date, and Period.
  4. Click Save.
Test DataRisk Owner: dewichatarina, PIC: rini.hervina, Likelihood: Possible, Severity: Major, Action: Reduce
Expected ResultRisk is saved with Risk Number format RN-BU-2026-XXX. Data appears in Need Follow Up tab. Risk Owner receives an email notification.
StatusPASSED
cancel TC-002 - Negative Case: Mitigation Mandatory Validation
Test IDTC-ERM-RI-002
ScenarioSystem rejects saving a risk that requires mitigation but mitigation fields are left empty
ModuleRisk Input - BAU
Pre-conditionUser logged in as Risk Owner. Current control evaluation is available.
Test Steps
  1. Open Risk Input menu.
  2. Fill all fields except mitigation.
  3. Set Action = "Reduce" (not Adequate/Avoid).
  4. Click Save without adding any mitigation.
Test DataAction: Reduce, Mitigation: empty
Expected ResultSystem displays a validation error. Risk is not saved.
StatusPASSED
cancel TC-003 - Negative Case: Update Cancelled Mitigation
Test IDTC-ERM-MIT-001
ScenarioMitigation with Cancelled status cannot be edited by PIC
ModuleMitigation Update Required - PIC
Pre-conditionA risk exists with a mitigation already marked as Cancelled. User logs in as the PIC for that mitigation.
Test Steps
  1. Log in as PIC (e.g., rini.hervina).
  2. Open Mitigation Update Required menu.
  3. Search for the risk with the Cancelled mitigation (ref: RN-BU-2026-611).
  4. Attempt to edit the Cancelled mitigation fields.
Test DataRisk Number: RN-BU-2026-611, Mitigation status: Cancelled
Expected ResultAll Cancelled mitigation columns are blocked/disabled. PIC cannot make any changes.
StatusPASSED
warning TC-004 - Edge Case: Prorate Mitigation Plan
Test IDTC-ERM-MIT-002
ScenarioProrated actual mitigation progress calculation no longer includes cancelled mitigations
ModuleRisk Register - Registered / Mitigation Progress
Pre-conditionA BAU risk exists with multiple active mitigations. One mitigation has been cancelled by the Risk Owner. The PIC for another mitigation has already updated progress.
Test Steps
  1. Log in as Risk Owner.
  2. Cancel one mitigation on risk RN-BU-2026-623.
  3. Log in as the PIC for the active mitigation.
  4. Update progress for the active mitigation.
  5. Check the Actual Cumulative Average column in RR and Dashboard L1/L2.
Test DataRisk: RN-BU-2026-623, Mitigation count: 2, Status: 1 active - 1 cancelled
Expected ResultProrated actual calculation only includes active mitigations. Cancelled mitigation is excluded from the divisor. Average values in RR and Dashboard remain consistent.
StatusPASSED
