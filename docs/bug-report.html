Laporan nyata dari kegagalan cron job pada sistem PostgreSQL.

Title            : [ERM-CRON-001] Cron Job mail-mitigation-reminder gagal dieksekusi
                   akibat inkompatibilitas fungsi DATE_FORMAT() di PostgreSQL

Application      : ERM V3
Environment      : Dev (ermdev.abm-investama.com)
Browser/OS       : N/A (Server-side Cron)
Tester           : Irfan Arimiwandi
Date Found       : 2026-06-09 (Date Issue ref: 46196)
Severity         : Critical
Priority         : P1
Status           : Open

Steps to Reproduce:
  1. Login ke server ERM V3.
  2. Jalankan cron job endpoint:
     GET /api/mail/mitigation/reminder
  3. Observasi response atau log server.

Expected Result  :
  Cron job berjalan sukses. Email reminder mitigation update
  terkirim ke seluruh PIC yang belum melakukan update pada
  periode berjalan.

Actual Result    :
  Cron job gagal dengan error:
  "Undefined function DATE_FORMAT"
  Database yang digunakan adalah PostgreSQL, namun query pada
  cron job menggunakan fungsi DATE_FORMAT() yang merupakan
  sintaks MySQL. Di PostgreSQL, fungsi yang valid adalah
  TO_CHAR(). Akibatnya, seluruh email reminder mitigation
  tidak terkirim ke pengguna.

Impact           :
  Seluruh PIC tidak menerima notifikasi reminder. Risk Owner
  tidak aware bahwa ada mitigasi yang belum diupdate. Flagging
  tetap merah tanpa ada pengingat otomatis, berpotensi
  menyebabkan compliance gap pada laporan risiko.

Attachments      : server_error_log_cron_46196.txt
Notes            :
  - Ditemukan juga error serupa pada cron lain di batch yang
    sama (mail-earlywarning-notif, mail-riskresidual-reminder).
  - Fix: ganti DATE_FORMAT(col, '%Y-%m') menjadi
    TO_CHAR(col, 'YYYY-MM') pada semua query cron.
  - Ref internal tester: "test by irf 11:13" pada sheet ERM V3.
