bug_report Critical Bug Example from Real Data

Actual report of a cron job failure in the PostgreSQL system.

Title            : [ERM-CRON-001] Cron Job mail-mitigation-reminder failed to execute
                   due to DATE_FORMAT() function incompatibility in PostgreSQL

Application      : ERM V3
Environment      : Dev (ermdev.abm-investama.com)
Browser/OS       : N/A (Server-side Cron)
Tester           : Syefdi Syaban
Date Found       : 2026-06-09 (Date Issue ref: 46196)
Severity         : Critical
Priority         : P1
Status           : Open

Steps to Reproduce:
  1. Login to the ERM V3 server.
  2. Trigger the cron job endpoint:
     GET /api/mail/mitigation/reminder
  3. Observe the response or server log.

Expected Result  :
  Cron job executes successfully. Mitigation update reminder emails
  are sent to all PICs who haven't updated in the current period.

Actual Result    :
  Cron job fails with error:
  "Undefined function DATE_FORMAT"
  The database used is PostgreSQL, but the query in the
  cron job uses the DATE_FORMAT() function, which is MySQL
  syntax. In PostgreSQL, the valid function is TO_CHAR().
  Consequently, all mitigation reminder emails fail to send.

Impact           :
  All PICs do not receive reminder notifications. Risk Owners
  are unaware of pending mitigation updates. Flags remain red
  with no automated reminders, potentially causing compliance
  gaps in risk reporting.

Attachments      : server_error_log_cron_46196.txt
Notes            :
  - Found similar errors in other cron jobs within the same
    batch (mail-earlywarning-notif, mail-riskresidual-reminder).
  - Fix: replace DATE_FORMAT(col, '%Y-%m') with
    TO_CHAR(col, 'YYYY-MM') in all cron queries.
  - Internal tester ref: "test by syefdi 11:13" on ERM V3 sheet.
